# toolbox — Claude Code ステアリングルール

## 必須ルール（例外なし）

### 1. AUDIT.md への変更記録
ファイルを変更・作成・削除するたびに、**コミット前に** [AUDIT.md](AUDIT.md) へエントリを追記すること。

フォーマット：
```
## YYYY-MM-DD HH:MM — <変更の概要（一行）>

- <変更ファイル>: <何をしたか>
- <変更ファイル>: <何をしたか>
```

### 2. 変更後は必ずコミット
ひとまとまりの作業が終わったら、必ず `git add` → `git commit` を実行すること。

コミットメッセージ形式：
- `feat: <新機能>` — 新しい機能の追加
- `fix: <修正内容>` — バグ修正
- `refactor: <内容>` — 機能変更を伴わないリファクタリング
- `chore: <内容>` — ビルド・設定・ドキュメント等の変更

Stop hookによる自動コミットが設定されているが、意味のある単位で手動コミットも行うこと。

- [kanban-board/](kanban-board/) — サーバー不要のシンプルなかんばんツール（単一HTMLファイル）

## コーディング規約

- 単一HTMLファイル構成を維持する（JS/CSSを外部ファイルに分離しない）
- コメントは「なぜ」が自明でない場合のみ追記する
- 変数名・関数名は英語、UIテキストは日本語
